## Commands Overview

```bash title="Usage"
mem-oracle <command>
```

If you're running from source, replace `mem-oracle` with `bun run src/index.ts`.

| Command | Description |
|---------|-------------|
| `worker` | Start the worker HTTP service |
| `serve` | Alias for `worker` |
| `mcp` | Start the MCP server (stdio) |
| `index` | Index a documentation URL |
| `search` | Search indexed documentation |
| `status` | Show indexing status |

---

## worker

Start the worker HTTP service that handles indexing and retrieval.

```bash title="Terminal"
mem-oracle worker
```

### Example

```bash title="Terminal"
# Start on default port
mem-oracle worker
```

---

## mcp

Start the MCP (Model Context Protocol) server for explicit tool calls.

```bash title="Terminal"
mem-oracle mcp
```

This starts an MCP server using stdio for communication. Use this with Claude Code's MCP configuration.

---

## index

Index a documentation website.

```bash title="Terminal"
mem-oracle index <url>
```

### Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `url` | Yes | The documentation URL to index |

### Examples

```bash title="Terminal"
# Index Next.js docs
mem-oracle index https://nextjs.org/docs/getting-started
```

---

## search

Search through indexed documentation.

```bash title="Terminal"
mem-oracle search <query>
```

### Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `query` | Yes | The search query |

### Examples

```bash title="Terminal"
# Basic search
mem-oracle search "how to use server components"
```

---

## status

Show the status of indexed documentation.

```bash title="Terminal"
mem-oracle status
```

### Output

```text title="Output"
Next.js Docs (https://nextjs.org)
  ID: abc123
  Status: ready
  Pages: 245/245 indexed
  Pending: 0, Errors: 0, Skipped: 0
  Chunks: 4120
```
